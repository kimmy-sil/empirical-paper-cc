### DAG 文字模板

```
节点：X（处理变量）、Y（结果变量）、Z（工具变量/外生冲击）、
      W1、W2（控制变量，前定）、M（机制变量，后定）、
      U（不可观测混淆因子）

箭头（因果关系）：
  Z → X          （工具变量影响处理）
  X → Y          （主效应路径）
  X → M → Y      （机制路径）
  W1 → X, W1 → Y （混淆，可控制）
  U → X, U → Y   （不可观测混淆，需识别策略处理）

禁止控制：
  M（后定变量，中介，控制会阻断因果路径）
  [对撞变量，如有]（控制会引入新的选择偏误）
```

---

### 识别假设模板

**DID：**
- 平行趋势：无处理时，处理组和对照组的 Y 趋势平行
- SUTVA：个体处理不影响他人结果（无溢出）
- 无预期效应：处理分配事先不知晓

**IV：**
- 相关性：Z 与 X 强相关（F > 10）
- 排他性：Z 只通过 X 影响 Y
- 独立性：Z 与 U 不相关

**RDD：**
- 连续性：潜在结果在断点处连续
- 无精确操控：个体无法精确控制运行变量跨越断点

## 填写示例：数据要素市场化试点对地区创新的影响

| 字段 | 内容 |
|------|------|
| **研究问题** | 数据要素市场化综合试验区试点政策对地级市专利申请量（创新产出）的因果效应 |
| **DAG（文字）** | 节点：Policy（试点）、Patent（专利数）、TechFirm（科技企业数）、GDPpc（人均GDP）、U（城市异质能力）；箭头：Policy→Patent（主效应），Policy→TechFirm→Patent（机制），GDPpc→Policy（选择偏误风险），U→Policy, U→Patent（不可观测混淆）；禁止控制：TechFirm（后定，中介变量） |
| **识别策略** | 交错DID：利用2020-2022年试点城市分批次设立的准实验变异。外生性来源：试点批次由国家数据局行政决定，非城市自选；批次间有时间差，且已有研究表明试点城市并非基于专利水平选择（预期效应弱）。 |
| **Estimand** | ATT（Average Treatment Effect on the Treated）：受试点政策影响的城市，相比未受政策影响时，专利申请量的平均变化。形式化：ATT = E[Y(1) - Y(0) \| D=1] |
| **数据** | 来源：国家知识产权局专利数据库 + 国家数据局试点名单 + 中国城市统计年鉴；样本：2016-2023年，278个地级市（含直辖市下属区）；观测数：约2200个城市-年度观测 |
| **方法路径** | Skill：did-analysis；软件：Python（linearmodels/pyfixest）+ R（fixest, did）；具体：TWFE基准→Bacon分解（确认负权重比例）→Callaway-Sant'Anna（2021）估计量→Sun-Abraham（2021）交叉验证→安慰剂500次→HonestDiD敏感性 |
| **文献贡献** | (1) 在数据上：使用专利申请微观数据而非宏观创新指数，匹配至城市层面，拓展了戚聿东等(2022)的省级分析；(2) 在方法上：采用CS估计量处理交错处理问题，克服Goodman-Bacon(2021)指出的TWFE偏误，改进了孙传尧等(2023)的TWFE回归；(3) 在机制上：通过DAG明确科技企业集聚为核心中介渠道，排除了政府数字化投入的替代解释。 |
| **风险 + Plan B** | 风险1：试点城市并非随机选择，可能存在选择偏误（高创新城市优先试点）→ Plan B：剔除首批试点城市，仅用第二、三批变异；补充PSM-DID检查样本可比性。风险2：预趋势不平行（部分城市试点前已有创新政策）→ Plan B：HonestDiD分析预趋势偏离的容忍度；分城市类型子样本分析。风险3：数据要素政策与其他数字经济政策（数字中国/智慧城市）同期 → Plan B：控制同期政策虚拟变量；剔除同年获得多个数字政策城市。 |
